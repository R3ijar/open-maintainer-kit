# Roadmap

## 0.1

- CSV-first maintainer triage.
- Release-readiness checks.
- Codex-ready prompt generation.
- Local privacy scan.
- Dogfooding on Open Growth Loop.

## 0.2

- Optional issue-draft export as Markdown.
- Stale PR and stale issue windows.
- Outcome memory for repeated maintainer decisions.
- More sample workspaces.

## 0.3

- Optional GitHub API import.
- Optional GitHub issue or PR comment draft creation.
- Release checklist templates.

## Principles

- Keep the core useful without accounts or secrets.
- Keep private data out of default inputs and reports.
- Prefer deterministic rules before model-generated recommendations.
